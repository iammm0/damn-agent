这个页面用于承载多 Agent 协作的基础模型。多 Agent 不是简单多开几个角色，而是要处理分工、通信、共享状态、并发和最终一致性。

## 建设边界

- 常见模式：manager-worker、handoff、debate、specialist team、blackboard。
- 角色设计：研究、规划、执行、评审、测试、总结。
- 通信机制：消息、共享状态、事件流、任务图、上下文转交。
- 并发控制：任务分片、结果合并、锁、取消、超时。
- 协作风险：重复劳动、角色漂移、信息丢失、循环争论。
- 适用边界：什么时候单 Agent 更简单，什么时候多 Agent 才值得。
