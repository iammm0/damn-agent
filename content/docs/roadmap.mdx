学习 Agent 最容易踩的坑，是一上来就陷入框架 API。更稳的路线是先理解系统结构，再进入实现细节。

## 阶段 1：建立心智模型

目标是把 Agent 看成一个可观察的控制循环。

- 输入：用户目标、环境状态、约束。
- 控制：规划、执行、检查、重试。
- 输出：结果、证据、轨迹、后续动作。

## 阶段 2：掌握核心概念

必须先理解这些概念：

| 概念 | 要回答的问题 |
| --- | --- |
| Agent Loop | 系统如何决定下一步做什么？ |
| Tool Calling | 模型如何安全使用外部能力？ |
| Memory | 哪些状态需要跨步骤或跨会话保留？ |
| Planning | 计划什么时候生成、什么时候更新？ |
| Evaluation | 如何判断 Agent 真的变好了？ |

## 阶段 4：选择框架

如果任务需要清晰状态机，优先看 LangGraph。如果你在 TypeScript 产品里集成 Agent 和 workflow，可以重点看 Mastra。如果你在研究多角色协作，再看 AutoGen 或 CrewAI。

## 阶段 5：拆解案例

案例学习要看五个问题：

1. 任务边界是什么。
2. 哪些步骤必须使用工具。
3. 状态如何保存。
4. 失败如何回放。
5. 什么情况下需要人工接管。
