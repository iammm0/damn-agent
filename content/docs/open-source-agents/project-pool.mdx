这个页面先维护项目池，不直接给质量结论。具体项目进入正文前，需要先完成仓库、许可证、运行方式和维护状态核验。

## 状态定义

| 状态 | 含义 |
| --- | --- |
| 待核验 | 已被提名，但还没有确认仓库、许可证、运行方式和维护状态。 |
| 可拆解 | 已完成基础核验，适合进入对应分类页写正文。 |
| 观察中 | 有学习价值，但维护、许可、运行路径或定位存在不确定性。 |
| 暂缓 | 当前不适合写入主文档，等待新版本或更多证据。 |
| 淘汰 | 不再维护、不可运行、许可证不适合或工程价值不足。 |

## 候选分类

| 分类 | 关注点 | 后续承载位置 |
| --- | --- | --- |
| 代码智能体 | 仓库理解、代码编辑、测试、提交、PR、review | `open-source-agents/coding-agents` |
| 研究智能体 | 检索、引用、资料整理、报告生成、来源追踪 | `open-source-agents/research-agents` |
| 浏览器与计算机使用智能体 | 浏览器控制、桌面操作、VNC、页面状态、人工接管 | `open-source-agents/browser-computer-use-agents` |
| 通用任务智能体 | 多工具、多步骤、任务编排、Web UI、产品化尝试 | `open-source-agents/general-purpose-agents` |
| 框架型项目 | 抽象、SDK、workflow、agent runtime | 优先放入 `frameworks`，必要时在本板块引用 |

## 收录前检查

- 是否是完整项目，而不是只有 prompt 或单文件 demo。
- 是否能跑通 README 中的最小路径。
- 是否能看清核心目录结构和任务执行链路。
- 是否有明确许可证。
- 是否能判断最近维护状态。
- 是否有值得学习的工程设计，而不是只因为热度高。
