这个板块专门讲社区中的开源智能体项目。它和「框架工具」不同：框架页关注 API、抽象和选型；开源智能体页关注一个具体项目如何组织目标、工具、记忆、运行环境、交互界面、部署方式和贡献生态。

## 为什么单独成板块

- 开源智能体项目比框架更接近可运行产品，能看到真实目录结构、工具封装、UI、任务状态和部署方式。
- 同一个项目往往同时包含 prompt、工具、浏览器、终端、记忆、队列、评测、权限和前端交互，不适合只放进框架页。
- 社区项目变化快，需要独立维护项目池、更新时间、仓库状态和适用边界。
- 读者需要知道哪些设计能借鉴，哪些只是 demo，哪些已经适合二次开发。

## 收录原则

- 必须能找到公开仓库、许可证、README、运行方式和维护痕迹。
- 优先拆解有真实工程结构的项目，不优先收录只停留在 prompt demo 的项目。
- 每个项目都要标注最后核验时间，避免把过期状态当作当前事实。
- 只写可复核内容：仓库结构、核心模块、运行方式、依赖、限制、适用场景。
- 不把 star 数、热度或宣传语当作质量结论。
