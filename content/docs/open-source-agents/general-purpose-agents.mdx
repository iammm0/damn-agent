这个页面用于承载通用任务智能体项目。它们通常不是只做代码、研究或浏览器，而是尝试把多种工具、任务状态和界面组合成完整产品。

## 建设边界

- 任务模型：目标、计划、子任务、状态、停止条件。
- 工具编排：搜索、浏览器、文件、Shell、数据库、API。
- UI 与交互：对话、任务进度、artifact、人工确认。
- 长任务：队列、checkpoint、恢复、取消、超时。
- 产品化：部署、权限、日志、成本、用户反馈。
- 失败模式：目标漂移、工具滥用、状态混乱、不可复现。

## 需要回答的问题

- 它是否真的能完成端到端任务。
- 它的状态和工具调用是否可追踪。
- 它如何处理失败、取消和人工确认。
- 它适合作为产品底座还是学习样例。
