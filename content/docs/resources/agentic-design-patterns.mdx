《智能体设计模式》（Agentic Design Patterns）是本站 Agent 概念层的主要参考书。它用模式语言组织智能体设计，适合在接触框架和具体项目之前建立统一心智模型。

## 这本书解决什么

- 把 Agent 拆成可命名的模式，而不是零散技巧。
- 说明工具、记忆、规划、反思、多 Agent 各自解决什么问题。
- 给出 MCP、Guardrails、评估等工程化主题的入门框架。
- 帮助读者判断某个设计是模式问题，还是具体产品实现细节。

## 阅读顺序建议

1. 先读工具使用和规划，理解 Agent 最小闭环。
2. 再读记忆和多 Agent，理解状态与协作边界。
3. 最后读 MCP、Guardrails 和评估，进入工程化思维。

如果同时阅读 [《Claude Code Harness Engineering》](/docs/resources/harness-engineering-book)，建议先用本书建立模式语言，再用 Harness 书补 Session、Sandbox 和执行隔离。

## 学习练习

读完本书后，可以用一个开源项目做反向标注：

1. 找到项目里的工具定义和调用链路。
2. 标出哪里实现了 planning 或 ReAct。
3. 找到短期状态和长期记忆分别在哪里保存。
4. 判断是否存在多 Agent 或角色分工。
5. 检查 guardrails 是 prompt、代码校验、权限系统还是人工审核。

这个练习能把模式语言和真实代码连接起来。
