## 这本书解决什么

- 说明代码 Agent 产品的多层架构，而不只停留在 prompt 技巧。
- 拆解查询引擎如何承载 Agent Loop、流式事件和 tool call。
- 分析统一工具接口、专用文件工具和 Bash 安全边界。
- 介绍 Hook、MCP、子 Agent、上下文优化和性能设计。

## 阅读顺序建议

1. 先读架构总览和查询引擎，建立 Agent Loop 中心视角。
2. 再读工具系统和文件工具，理解为什么不用 Bash 包打天下。
3. 然后读子 Agent、Hook、MCP，看扩展边界。
4. 最后读上下文与性能，把「省 token」当成架构能力。

阅读前建议先有 [《Claude Code Harness Engineering》](/docs/resources/harness-engineering-book) 的 Session/Harness/Sandbox 概念；本书更贴近具体产品实现。
